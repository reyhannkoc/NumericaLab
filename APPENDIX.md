# 1. Lesson Acceptance Criteria

A lesson is considered complete only if ALL of the following exist.

## Educational Content

✓ Introduction

✓ Motivation

✓ Theory

✓ Mathematical Background

✓ Definitions

✓ Key Concepts

✓ Formula Explanation

---

## Visualization

At least one interactive visualization.

Students should manipulate parameters.

Graphs update immediately.

---

## Animation

At least one algorithm animation.

Animation supports:

- Play
- Pause
- Reset
- Step Forward
- Speed Control

---

## Playground

Students should freely experiment.

Playground must support:

- User input
- Parameter sliders
- Presets
- Reset
- Instant recalculation

---

## Algorithm Execution

Every iteration must be visible.

Display:

- Current iteration
- Current estimate
- Error
- Convergence

---

## Error Analysis

Explain:

- Floating Point Error
- Truncation Error
- Numerical Stability
- Limitations

---

## Performance Analysis

Display:

- Runtime
- Iterations
- Complexity
- Memory (if applicable)

---

## Comparison

If multiple methods exist:

Provide side-by-side comparison.

---

## Engineering Applications

Minimum:

Five realistic examples.

---

## Practice

Minimum:

Four exercises.

Difficulty:

- Easy
- Medium
- Hard
- Challenge

---

## Summary

Must include:

- Cheat Sheet
- Key Formulas
- Best Practices
- Next Lesson

---

# Lesson Completion Rule

Missing even ONE required section means the lesson is incomplete.

# 2. Module Acceptance Criteria

A module is complete only if:

✓ Every lesson complete

✓ Comparison page complete

✓ Overview page complete

✓ Laboratory integration complete

✓ Quiz complete

✓ Production build successful

---

# 3. Visualization Rules

Graphs should never be static.

Users should always interact.

Allowed interactions:

- Zoom
- Pan
- Hover
- Slider
- Click
- Drag (when meaningful)

---

# 4. Animation Rules

Animations should teach.

Avoid decorative animations.

Every animation must answer:

"What is the algorithm doing?"

---

# 5. Playground Rules

Changing a parameter should update:

Graph

↓

Table

↓

Animation

↓

Performance

↓

Error

Automatically.

Never require a page refresh.

---

# 6. Educational Quality Rules

Always explain:

Why?

How?

When?

When NOT?

Students should never memorize algorithms.

They should understand them.

---

# 7. Comparison Rules

Comparison pages must include:

- Accuracy
- Runtime
- Iterations
- Memory
- Convergence
- Stability
- Advantages
- Disadvantages
- Recommended Usage
